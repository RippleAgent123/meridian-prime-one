MERIDIAN ← DASHBOARD EMAIL ORGANIZER

Integration

Email Auto-Organizer

Connect Gmail or Outlook. Meridian creates a folder per transaction and routes incoming emails by address and client name — automatically.

GMAIL

Connects via Google OAuth 2.0. Requires Gmail API scope: gmail.modify. Folders created as Gmail labels.

OUTLOOK / MICROSOFT 365

Connects via Microsoft OAuth 2.0 (MSAL). Requires Mail.ReadWrite permission. Folders created as Outlook mail folders.

HOW IT WORKS

01You connect your email account (Gmail or Outlook) via OAuth. Meridian never stores your email password.
02For each active transaction in your Meridian file system, a folder is created: 8812 Oak Ave — Johnson Deal
03Incoming emails are scanned for matching property addresses, client names, and known parties (title company, lender, co-agent). Matches are moved automatically.
04Unmatched emails remain in your inbox. Meridian never moves emails it isn't confident about — a 90%+ confidence threshold applies.
05When a transaction closes, the folder is archived. The thread history stays intact and searchable.

Setup required before deployment: Register an OAuth app at console.cloud.google.com (Gmail) and portal.azure.com (Outlook). Set redirect URI to your Vercel deployment URL. Add GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET, OUTLOOK_CLIENT_ID, and OUTLOOK_CLIENT_SECRET to your environment variables. See /api/auth/gmail.js and /api/auth/outlook.js in the Meridian repo for the handler templates.
